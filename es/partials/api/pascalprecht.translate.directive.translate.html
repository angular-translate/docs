translate
directive in module pascalprecht.translate

Description

Translates given translation id either through attribute or DOM content. Internally it uses $translate service to translate the translation id. It possible to pass an optional translate-values object literal as string into translation id.

Dependencies

  • $interpolate,
  • $compile,
  • $parse,
  • $rootScope

Usage

as element:
<translate
       [translate="{string}"]
       [translate-values="{string}"]
       [translate-attr--a-t-t-r="{string}"]
       [translate-default="{string}"]
       [translate-compile="{boolean}"]
       [translate-keep-content="{boolean}"]>
</translate>
as attribute
<ANY translate="{string}"
     [translate-values="{string}"]
     [translate-attr--a-t-t-r="{string}"]
     [translate-default="{string}"]
     [translate-compile="{boolean}"]
     [translate-keep-content="{boolean}"]>
   ...
</ANY>

Parameters

ParamTypeDetails
translate
(optional)
string

Translation id which could be either string or interpolated string.

translate-values
(optional)
string

Values to pass into translation id. Can be passed as object literal string or interpolated object.

translate-attr-ATTR
(optional)
string

translate Translation id and put it into ATTR attribute.

translate-default
(optional)
string

will be used unless translation was successful

translate-compile
(optional)
boolean

(default true if present) defines locally activation of pascalprecht.translate.$translateProvider#methods_usePostCompiling

translate-keep-content
(optional)
boolean

(default true if present) defines that in case a KEY could not be translated, that the existing content is left in the innerHTML}
